## 4. 距离“可直接构建基线版”的最小缺口

在评审视角下，基线版当前已无主链级文档缺口。

如果后续目标从“基线版”提升为“完整扩展版”，更合理的追加方向是：

1. context collapse 的单列恢复协议。
2. 分布式运行时与远端 transport 的进一步规格化。
3. 更平台化的 browser / container / computer-use 宿主能力。

而 `local_workflow` / `monitor_mcp` 一类类型当前已在补遗中明确标注为保留槽位，不再属于“文档缺口”，而属于“功能未实装的扩展点”。
