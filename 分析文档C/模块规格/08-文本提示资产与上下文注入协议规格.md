# 文本提示资产与上下文注入协议规格

## 文档状态

- 状态：Draft
- 日期：2026-04-03
- 对应 Rust 目标模块：`agent-context-assets`、`agent-context`、`agent-code-workflows`

## 1. 目标

定义系统内所有“文本提示资产”的统一协议，使 Rust 版能够稳定复现：

1. system prompt、user context、skill/command、agent prompt、classifier prompt 的装配边界。
2. 不同来源文本资产的发现、优先级、覆盖与信任规则。
3. cache-safe prefix、compaction、resume 和 side-query 之间对提示资产的共享不变量。

## 2. 非目标

本规格明确不覆盖：

1. 具体 prompt 文案内容本身。
2. 任何产品 marketing 文案或 UI 标签。
3. 远端服务 API 细节和 OAuth 协议。
4. review / PR / planning 等工作流的业务逻辑细节。

## 3. 设计原则

### 3.1 文本资产必须按注入平面建模

同样是 markdown 或 txt，可能进入：

1. 主会话 system prompt。
2. user context。
3. meta user message。
4. agent-local system prompt。
5. classifier / maintenance side-query。

Rust 版必须显式建模这些平面，不能只存 `String`。

### 3.2 discovery source 和 injection plane 必须分离

同一注入平面可以来自多种 source：

1. built-in code strings。
2. bundled build-time 文档。
3. managed/user/project/local 文件系统。
4. plugin 包内容。
5. MCP 远端内容。
6. CLI flag / file 覆盖。

source 决定信任与优先级；plane 决定装配位置。两者不能混用。

### 3.3 覆盖与追加必须是不同操作

至少要区分：

1. 替换默认 system prompt。
2. 在默认 system prompt 后追加文本。
3. 向 user context 增加指令块。
4. 向 transcript 注入一条 meta user message。
5. 在 agent-local system prompt 尾部追加 memory prompt。

### 3.4 trust boundary 必须进入 prompt 装配协议

本地 user/project 资产、plugin 资产、MCP 远端资产、内部生成资产的安全假设不同。

典型约束包括：

1. 本地 skill 可执行 shell interpolation。
2. MCP skill 不可执行 shell interpolation。
3. plugin agent 不能自带 `permissionMode` / `hooks` / `mcpServers`。
4. classifier 可读取 `CLAUDE.md`，但不应继承主对话的任意 tool side effects。

### 3.5 cache boundary 必须由 prompt assembler 统一维护

`SYSTEM_PROMPT_DYNAMIC_BOUNDARY` 体现出 system prompt 前缀缓存不是 API 层的偶然优化，而是装配协议的一部分。

Rust 版必须能在 asset 级别标记 cache scope，而不是在字符串拼接后再猜测边界。

## 4. 文本资产分类

建议将文本资产分为以下七类：

1. `MainSystemPromptAsset`
2. `UserInstructionAsset`
3. `WorkflowPromptAsset`
4. `AgentPromptAsset`
5. `OutputStyleAsset`
6. `MaintenancePromptAsset`
7. `ClassifierPromptAsset`

### 4.1 MainSystemPromptAsset

来源示例：

1. `constants/prompts.ts`
2. `main.tsx` 提供的 `--system-prompt` / `--system-prompt-file`
3. `--append-system-prompt` / `--append-system-prompt-file`

注入平面：主会话 system prompt。

### 4.2 UserInstructionAsset

来源示例：

1. `CLAUDE.md`
2. `.claude/CLAUDE.md`
3. `.claude/rules/*.md`
4. memory 入口文件与 include 展开内容

注入平面：主会话 user context，必要时复用于 classifier。

### 4.3 WorkflowPromptAsset

来源示例：

1. `.claude/skills/*/SKILL.md`
2. plugin commands/skills markdown
3. bundled skill 正文
4. 远端 canonical skill `SKILL.md`

注入平面：meta user message。

### 4.4 AgentPromptAsset

来源示例：

1. `.claude/agents/*.md`
2. plugin agent markdown

注入平面：agent-local system prompt。

### 4.5 OutputStyleAsset

来源示例：

1. built-in output styles
2. `.claude/output-styles/*.md`
3. plugin output style markdown

注入平面：主会话 system prompt 的 output-style section，并附带 transcript attachment。

### 4.6 MaintenancePromptAsset

来源示例：

1. session memory `template.md` / `prompt.md`
2. magic docs `prompt.md`
3. auto-dream consolidation prompt
4. ultraplan `prompt.txt`

注入平面：内部维护型子代理或专用 workflow。

### 4.7 ClassifierPromptAsset

来源示例：

1. `auto_mode_system_prompt.txt`
2. `permissions_external.txt`
3. `permissions_anthropic.txt`

注入平面：权限 classifier 的 side-query system prompt。

## 5. 通用数据模型

建议定义：

```rust
pub enum PromptInjectionPlane {
    MainSystem,
    UserContext,
    MetaUserMessage,
    AgentSystem,
    SideQuerySystem,
    Attachment,
}

pub enum PromptAssetSource {
    BuiltIn,
    Bundled,
    ManagedFs,
    UserFs,
    ProjectFs,
    LocalFs,
    Plugin,
    Mcp,
    CliInline,
    CliFile,
    InternalGenerated,
}

pub enum PromptTrustLevel {
    TrustedManaged,
    TrustedLocal,
    PluginInstalled,
    RemoteUntrusted,
    InternalOnly,
}

pub enum PromptCacheScope {
    Global,
    Session,
    None,
}

pub struct PromptAsset {
    pub id: String,
    pub source: PromptAssetSource,
    pub plane: PromptInjectionPlane,
    pub trust: PromptTrustLevel,
    pub text: String,
    pub cache_scope: PromptCacheScope,
    pub replaces_default: bool,
    pub appends_to_tail: bool,
    pub allow_shell_expansion: bool,
    pub visible_to_user: bool,
    pub priority: i32,
}
```

## 6. 发现与优先级规则

### 6.1 主会话 system prompt

主会话 system prompt 的装配顺序应为：

1. 若存在 `customSystemPrompt`，则跳过默认 `getSystemPrompt()` 构建。
2. 否则使用 `constants/prompts.ts` 的默认 section 集合。
3. 视条件插入 `memoryMechanicsPrompt` 之类的 runtime addendum。
4. 末尾追加 `appendSystemPrompt`。

### 6.2 `CLAUDE.md` / rules / memory

`UserInstructionAsset` 应按以下顺序发现并保留优先级：

1. managed
2. user
3. project
4. local
5. 更靠近当前工作目录的 project/local 文件优先级更高

并支持：

1. `@include` 展开
2. frontmatter path 过滤
3. 循环引用保护
4. 长入口文件截断

### 6.3 `.claude/<subdir>` 资产

`commands`、`agents`、`output-styles` 这类资产应通过统一目录发现器加载：

1. 从当前目录向上到 git root，避免仓库外父目录泄漏。
2. 支持 worktree fallback。
3. 对 inode 去重。
4. 明确区分 managed/user/project source。

### 6.4 bundled 资产

bundled 文本应被视为独立 source，而不是伪装成普通文件读取：

1. 构建期内嵌，无运行时路径依赖。
2. 仍需保留 asset id 与 logical path，便于调试和测试。

## 7. 注入协议

### 7.1 主会话 system prompt 注入

装配器必须输出结构化 blocks，而不是单纯字符串：

1. 先按 `PromptCacheScope` 分块。
2. 在 `SYSTEM_PROMPT_DYNAMIC_BOUNDARY` 前后的块使用不同 cache scope。
3. 送入 API 前保留 block 边界。

### 7.2 user context 注入

`CLAUDE.md` 一类资产不应并入 default system prompt 文本，而应进入 `userContext`。

其原因：

1. 与 system prompt 的来源与优先级不同。
2. 需要被 classifier 复用。
3. 更适合被视为“用户补充意图”而非平台 hardcode。

### 7.3 workflow / skill 注入

workflow prompt 的正确注入方式应为：

1. 生成 command-loading metadata。
2. 将 workflow 正文作为 `meta user message` 注入 transcript。
3. 附加 `command_permissions` attachment。
4. 记录 invoked skill 供 compaction 恢复。

禁止把 skill 正文直接拼到主 system prompt。

### 7.4 agent prompt 注入

agent markdown 应被解析为：

1. frontmatter 元数据
2. system prompt body
3. 可选 memory addendum

agent prompt 只作用于该 agent 的 query runtime，不得回写主会话 default system prompt。

### 7.5 classifier / maintenance prompt 注入

classifier、session-memory、magic-docs、auto-dream 一类资产必须走独立 side-query/maintenance assembler：

1. 与主会话 tool pool 隔离。
2. 与主会话 transcript 仅通过显式投影共享信息。
3. 可拥有自己专用 system prompt 模板。

## 8. 安全与信任策略

### 8.1 shell interpolation 规则

仅以下 source 默认允许 shell interpolation：

1. trusted local skills
2. trusted project/user/plugin markdown command bodies

以下 source 默认禁止：

1. MCP skills
2. 远端 canonical skills
3. classifier / maintenance prompt templates

### 8.2 插件 agent 限制

plugin agent 虽然是 `AgentPromptAsset`，但不得声明：

1. `permissionMode`
2. `hooks`
3. `mcpServers`

这类限制应在 parser 阶段而不是执行阶段完成。

### 8.3 classifier 与主会话隔离

classifier 可以读取：

1. 投影后的 transcript
2. `CLAUDE.md`
3. action compact

但不应继承：

1. 主会话 workflow prompt 正文
2. 任意本地 shell interpolation 副作用
3. agent-local system prompt

## 9. compaction 与恢复规则

### 9.1 skill / workflow 注入必须可恢复

只要 workflow 正文被注入 transcript，就必须记录：

1. skill name
2. skill source/path
3. skill content
4. 所属 agentId

否则 compaction 后无法重建同一工作流上下文。

### 9.2 output style 需要双重保真

output style 应同时通过两种方式保真：

1. system prompt section
2. transcript attachment

这样在 compact / resume 后仍能知道原会话启用了非默认输出风格。

### 9.3 user context 需要可复投影

`CLAUDE.md` 等 user context 资产应作为可重复构建的 projection，而不是一次性拼接后丢失来源。

## 10. Rust 组装建议

建议新增三层接口：

```rust
pub trait PromptAssetDiscovery {
    async fn discover(&self, req: DiscoveryRequest) -> Vec<PromptAsset>;
}

pub trait PromptTrustPolicy {
    fn normalize(&self, asset: PromptAsset) -> PromptAsset;
}

pub trait PromptAssembler {
    async fn assemble_main_loop(&self, req: MainLoopPromptRequest) -> MainLoopPrompt;
    async fn assemble_agent(&self, req: AgentPromptRequest) -> AgentPrompt;
    async fn assemble_side_query(&self, req: SideQueryPromptRequest) -> SideQueryPrompt;
    async fn assemble_workflow(&self, req: WorkflowPromptRequest) -> WorkflowPrompt;
}
```

其中：

1. `PromptAssetDiscovery` 只负责找资产。
2. `PromptTrustPolicy` 负责 source 到 trust policy 的映射与裁剪。
3. `PromptAssembler` 负责按 plane 和 precedence 形成最终注入结果。

## 11. 最小测试矩阵

### 11.1 system prompt

1. `customSystemPrompt` 应替换默认 prompt。
2. `appendSystemPrompt` 应稳定追加到尾部。
3. dynamic boundary 前缀应获得全局 cache scope。

### 11.2 user context

1. `CLAUDE.md` / `.claude/rules` 应按优先级装载。
2. `@include` 应防止循环引用。
3. classifier 应能复用缓存的 `claudeMd`。

### 11.3 workflow assets

1. skill 正文应注入为 meta user message。
2. MCP skill 不应执行 shell interpolation。
3. invoked skill 信息应可用于 compaction 恢复。

### 11.4 agent assets

1. agent body 应成为 agent-local system prompt。
2. memory scope 启用时应追加 agent memory prompt。
3. plugin agent 中不允许的 frontmatter 字段应被忽略或告警。

### 11.5 output style / classifier

1. 非 default output style 应进入 system prompt section。
2. 非 default output style 应产生 attachment。
3. classifier prompt 模板切换应遵守 external vs anthropic 选择规则。

## 12. 结论

Rust 版真正需要继承的不是一堆 prompt 文案，而是一套“文本资产协议”：

1. 发现规则
2. 优先级规则
3. 注入平面
4. 缓存边界
5. trust boundary
6. 覆盖与恢复语义

只有把这些规则独立建模，后续 `agent-context`、`agent-code-workflows`、`agent-subagents` 才不会在实现阶段互相污染。
