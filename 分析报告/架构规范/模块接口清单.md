# 模块接口清单

## 文档状态

- 状态：Proposed
- 日期：2026-04-02
- 用途：作为新一代 Rust Code Agent 的 crate 划分、公共接口和依赖边界基线

## 使用方式

这不是最终 API 文档，而是架构级接口清单。团队在开始实现每个 crate 前，应先对照本清单确认：

1. 模块职责是否越界。
2. 是否引入了反向依赖。
3. 公共接口是否足以支撑宿主层和其它模块。
4. 错误边界是否清晰。

## 模块总表

| 模块 | 核心职责 | 允许依赖 | 禁止依赖 |
| --- | --- | --- | --- |
| `agent-message` | 消息 IR、转录、规范化、序列化 | 标准库、共享类型 | UI、transport 宿主 |
| `agent-context` | prompt section、context layers、memory assembly、budget | `agent-message` | host、transport |
| `agent-query` | query state machine、stream event reduction、continuation | `agent-message` `agent-context` `agent-tools` `agent-permissions` | host-ui |
| `agent-tools` | tool descriptor、tool executor、tool scheduler、tool context | `agent-message` `agent-permissions` | host-ui、transport 宿主 |
| `agent-permissions` | permission mode、rule engine、risk analyzer | 共享类型 | host-ui |
| `agent-tasks` | background task runtime、task state、task events、output artifacts | `agent-message` `agent-query` `agent-tools` | host-ui |
| `agent-extensions` | MCP、skills、plugins、workflows 的标准化层 | `agent-message` `agent-tools` `agent-permissions` | host-ui |
| `agent-transport` | bridge、remote session、auth refresh、keepalive | `agent-message` | host-ui、query reducer 实现细节 |
| `agent-runtime` | bootstrap、config、composition root、mode wiring | 所有 `agent-*` 内核模块 | host-ui 具体组件 |
| `host-cli` | CLI 参数、stdout/stderr、shell lifecycle | `agent-runtime` | 内核反向依赖 |
| `host-tui` | TUI 展示与交互 | `agent-runtime` | 内核反向依赖 |
| `host-sdk` | 外部程序嵌入式 API | `agent-runtime` | 内核反向依赖 |

## 1. agent-message

### 职责

- 定义统一消息 IR
- 负责 transcript 的持久化模型
- 负责消息规范化和序列化
- 提供 tool result pairing、message filtering、compact boundary 标记基础设施

### 关键公开类型

```rust
pub struct SessionId(pub String);
pub struct TurnId(pub String);
pub struct MessageId(pub String);
pub struct ToolUseId(pub String);
pub struct TaskId(pub String);

pub enum Message {
    User(UserMessage),
    Assistant(AssistantMessage),
    System(SystemMessage),
    Progress(ProgressMessage),
    TaskNotification(TaskNotificationMessage),
    CompactBoundary(CompactBoundaryMessage),
}

pub enum MessageBlock {
    Text(TextBlock),
    ToolUse(ToolUseBlock),
    ToolResult(ToolResultBlock),
    Thinking(ThinkingBlock),
    Attachment(AttachmentBlock),
}
```

### 关键公开 trait

```rust
pub trait TranscriptStore {
    fn append(&self, session: &SessionId, messages: &[Message]) -> Result<(), TranscriptError>;
    fn load(&self, session: &SessionId) -> Result<Vec<Message>, TranscriptError>;
    fn flush(&self, session: &SessionId) -> Result<(), TranscriptError>;
}

pub trait MessageNormalizer {
    fn normalize_for_model(&self, messages: &[Message]) -> Result<Vec<ModelMessage>, NormalizeError>;
    fn ensure_tool_result_pairing(&self, messages: &[Message]) -> Result<Vec<Message>, NormalizeError>;
}
```

### 输入输出契约

- 输入：宿主输入事件、tool result、remote message、task notification
- 输出：统一 Message 列表、标准化模型消息、可持久化 transcript 记录

### 错误边界

- `NormalizeError`
- `TranscriptError`
- `SerializationError`

## 2. agent-context

### 职责

- 构建系统提示与上下文层
- 管理 memory inventory 与相关记忆装配
- 管理预算和 compact 触发判断
- 维护 cache-safe prefix

### 关键公开类型

```rust

    pub user_layers: Vec<ContextLayer>,
    pub system_layers: Vec<ContextLayer>,
    pub memory_artifacts: Vec<MemoryArtifact>,
    pub cache_safe_prefix: CacheSafePrefix,
}

pub struct BudgetState {
    pub context_window: usize,
    pub reserved_output_tokens: usize,
    pub warning_threshold: usize,
    pub blocking_threshold: usize,

```

### 关键公开 trait

```rust

    fn build(&self, request: &ContextBuildRequest) -> Result<ContextBundle, ContextError>;

pub trait MemorySelector {
    fn select(&self, query: &str, available: &[MemoryHeader]) -> Result<Vec<MemoryArtifact>, MemoryError>;

    fn decide(&self, transcript: &TranscriptView, budget: &BudgetState) -> CompactionDecision;
```

## 3. agent-query

### 职责

- 维护单次回合的状态机
- 驱动模型流式采样与工具回注
- 管理 stop hooks、continuation、fallback 与终止条件

### 关键公开类型


```rust

```

- `ModelClient`
- `ToolExecutor`
- `PermissionEvaluator`
- `TranscriptStore`
- `CompactionPolicy`

## 10. host-cli / host-tui / host-sdk

### 职责边界

- 仅负责用户交互与平台适配
- 不得直接修改 query 状态机内部状态
- 不得自行定义新的权限语义或工具调度规则

### 推荐对内接口

```rust
pub trait HostAdapter {
    fn submit_prompt(&mut self, prompt: PromptInput) -> Result<(), HostError>;
    fn poll_events(&mut self) -> Result<Vec<HostEvent>, HostError>;
    fn render_messages(&mut self, messages: &[Message]) -> Result<(), HostError>;
}
```

## 跨模块共享类型

建议单独维护 `agent-types` 或放在 `agent-message`/`agent-runtime` 中的共享域：

- `SessionId`
- `MessageId`
- `TaskId`
- `ToolUseId`
- `RuntimeMode`
- `ModelRef`
- `ArtifactRef`

## 依赖约束

### 必须遵守

1. `agent-message` 不依赖其它业务模块。
2. `agent-query` 不依赖任何宿主层 crate。
3. `agent-transport` 只通过 `Message` 和显式 envelope 与内核通信。
4. `host-*` 只能依赖 `agent-runtime` 暴露的稳定接口。

### 必须避免

1. `host-*` 反向依赖进入 `agent-query` 内部状态结构。
2. `agent-extensions` 直接持有宿主层 UI 对象。
3. `agent-tools` 直接读写 transport 内部连接状态。

## 接口冻结建议

在进入 Phase 1 开发前，建议优先冻结以下接口：

1. `Message`
2. `QueryEvent`
3. `QueryState`
4. `ToolCall`
5. `PermissionDecision`
6. `TaskRecord`
7. `CapabilityDescriptor`
8. `RemoteEnvelope`

这八类接口构成最小架构骨架，一旦频繁变化，会导致各模块并行推进困难。
