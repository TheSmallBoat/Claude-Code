# 扩展-传输-多代理实现设计

## 文档状态

- 状态：Proposed
- 日期：2026-04-02
- 对应上游：ADR-005、ADR-006、模块接口清单

## 目标

定义 extensions、transport、remote session、subagent、coordinator 的实现组织方式，使系统在不破坏本地内核稳定性的前提下接入平台化与分布式能力。

## Extension Layer 设计

建议 `agent-extensions` 只做标准化，不做最终执行。它的职责是把 MCP、skills、plugins、workflows 装配为统一能力对象，再交给 tools 或 runtime 消费。

建议核心接口：

```rust
pub trait ExtensionRegistry {
    fn load_all(&self, ctx: &ExtensionLoadContext) -> Result<Vec<ExtensionCapability>, ExtensionError>;
}
```

`ExtensionCapability` 应至少声明：

- capability id
- source kind
- exported tools
- exported commands
- permission hints
- lifecycle hooks

## MCP 接入原则

- 协议适配器与业务语义分离
- stdio、SSE、HTTP streaming、WebSocket 只属于 transport adapter 层
- 暴露给 runtime 的应是统一 `CapabilityProvider`

## Transport Layer 设计

`agent-transport` 应只负责会话连接、认证、保活、消息发送与接收，不负责 query reducer 逻辑。

建议接口：

```rust
pub trait TransportManager {
    fn open_session(&self, req: OpenTransportSession) -> Result<TransportSessionHandle, TransportError>;
    fn reconnect(&self, session_id: &SessionId) -> Result<(), TransportError>;
}
```

## Remote Session 模型

远程会话建议采用“双层对象”：

1. 本地 `SessionRuntime`
2. 外层 `RemoteAttachment`

这样可以避免把远程连接状态直接塞进 query 内核。

## Subagent 与 Coordinator

建议统一把 subagent、background agent、coordinator worker 视为 `AgentInstance` 的不同 mode：

```rust
pub enum AgentMode {
    Interactive,
    BackgroundTask,
    ForkedSubagent,
    CoordinatorWorker,
    RemoteAttached,
}
```

所有 mode 共用：

- 消息 IR
- query runtime
- transcript store
- permission kernel

只有 host adapter、transport attachment、resource budget 不同。

## 多代理通信原则

- agent 之间统一通过消息或 artifact 引用交互
- 不允许跨 agent 共享内部可变状态
- coordinator 只下发任务与汇总结果，不直接操作子代理 reducer

## 实现顺序

1. 先完成本地 extension registry。
2. 再接 MCP provider adapter。
3. 然后实现最小 remote session attach。
4. 最后实现 forked subagent 与 coordinator worker。
