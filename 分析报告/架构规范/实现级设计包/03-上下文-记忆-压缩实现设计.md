# 上下文-记忆-压缩实现设计

## 文档状态

- 状态：Proposed
- 日期：2026-04-02
- 对应上游：ADR-003、模块接口清单

## 目标

把上下文系统从“提示词拼接”提升为“预算化上下文装配管线”，明确 prompt sections、memory selection、session memory、compaction 与 cache-safe prefix 的实现职责和数据流。

## 核心流水线

建议 `agent-context` 实现为固定装配流水线：

```text
+------------------------+
| Prompt Sections        |
+------------------------+
           |
           v
+------------------------+
| Static Context Layers  |
+------------------------+
           |
           v
+------------------------+
| Memory Selection       |
+------------------------+
           |
           v
+------------------------+
| Budget Evaluation      |
+------------------------+
           |
           v
+------------------------+
| Compaction Decision    |
+------------------------+
           |
           v
+------------------------+
| Context Bundle         |
+------------------------+
```

## 子模块划分

建议至少包含以下子模块：

- `prompt_sections`：系统提示片段注册、排序、cache-breaking 标注
- `memory_catalog`：记忆头信息、来源、标签、时间戳、权重
- `memory_selector`：按 query 与 session 选择相关记忆
- `session_memory`：会话级摘要、近期状态、压缩摘要
- `budget_engine`：上下文窗口预算、阈值判断、保留输出预算
- `compaction_engine`：触发策略、摘要生成、摘要替换
- `cache_prefix`：稳定前缀构建与缓存安全控制

## Prompt Section 原则

- section 必须具备稳定 id，不能只靠自由文本定位
- section 顺序必须可预测
- cache-breaking section 必须显式标注
- host-specific section 不得混入 kernel 级 section

建议结构：

```rust
pub struct PromptSectionSpec {
    pub id: String,
    pub priority: u16,
    pub cache_breaking: bool,
    pub render: Arc<dyn SectionRenderer>,
}
```

## Memory Selection 策略

记忆选择建议分三步：

1. 预筛：按 scope、session、workspace、标签过滤。
2. 相关性排序：按 query、时间衰减、人工 pin 权重排序。
3. 预算裁剪：在 token 预算内截断或摘要化。

选择器不应直接读取原始文件系统，而应通过抽象存储接口获取记忆元数据和内容。

## Budget Engine

预算引擎应返回的不只是“是否超限”，而是一组明确决策：

```rust
pub enum BudgetDecision {
    Allow,
    Warn(BudgetWarning),
    Compact(CompactionPlan),
    Reject(BudgetOverflow),
}
```

## Compaction 约束

- compaction 必须保留 tool result pairing 所需语义
- 压缩产物必须具备来源边界与版本号
- summary 不得伪造用户未表达的事实
- 被压缩片段与生成摘要之间必须能建立引用关系

建议结构：

```rust
pub struct CompactionArtifact {
    pub artifact_id: String,
    pub source_range: TranscriptRange,
    pub summary_message: Message,
    pub version: u32,
}
```

## Cache-safe Prefix

为提高缓存稳定性，上下文构建结果应显式区分：

- stable prefix
- volatile suffix

只有 stable prefix 才允许进入跨回合缓存。任何 session-specific、tool-specific、volatile timestamp 内容都不得混入 stable prefix。

## 落盘边界

建议持久化以下对象：

- memory catalog index
- session memory snapshot
- compaction artifact
- context assembly metadata

不建议持久化每次完整 `ContextBundle`，而应持久化可重建元数据。

## 测试要求

至少应覆盖：

1. 大量记忆候选下的预算裁剪。
2. 压缩前后 tool pairing 语义保持。
3. cache-safe prefix 稳定性。
4. session resume 后的上下文重建一致性。
